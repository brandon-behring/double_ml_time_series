OJ Dataset (dml_ts.data.oj_loader)#

Dominick’s orange juice scanner data — a standard cross-sectional causal inference benchmark dataset.

Dominick’s Orange Juice Dataset Loader.

Loads the OJ dataset from Microsoft Azure blob storage. This dataset contains store-level panel data on orange juice sales from Dominick’s Finer Foods, a major Chicago-area supermarket chain.

The dataset is widely used for causal inference demonstrations, particularly for estimating price elasticity of demand with confounders like income and demographics.

Data Source#

University of Chicago Booth School of Business, Kilts Center for Marketing. https://research.chicagobooth.edu/kilts/marketing-databases/dominicks

GitHub mirror (used here): gchoi/Dataset

References

Chernozhukov, V., et al. (2018). Double/debiased machine learning for treatment and structural parameters. The Econometrics Journal, 21(1), C1-C68.

Montgomery, A. (1997). Creating Micro-Marketing Pricing Strategies Using Supermarket Scanner Data. Marketing Science, 16(4), 315-337.

class dml_ts.data.oj_loader.OJDataset[source]#

Bases: object

Container for Orange Juice dataset prepared for DML analysis.

This dataclass holds the outcome, treatment, and confounders in a format ready for DML estimation. The dataset is preprocessed to match standard econometric conventions.

Variables:

  • Y (NDArray[np.float64]) – Outcome variable: log(units sold) = logmove. Shape: (n_samples,)

  • T (NDArray[np.float64]) – Treatment variable: log(price). Shape: (n_samples,)

  • X (NDArray[np.float64]) – Confounder matrix: [feat, INCOME, AGE60, …]. Shape: (n_samples, n_features)

  • feature_names (list[str]) – Names of confounders in X columns.

  • n_samples (int) – Number of observations.

  • n_features (int) – Number of confounders.

  • raw_df (pd.DataFrame) – Original dataframe before transformation (for reference).

Examples

>>> loader = OJDataLoader()
>>> data = loader.load()
>>> print(f"Samples: {data.n_samples}, Features: {data.n_features}")
Samples: 28947, Features: 3
>>> from dml_ts.dml import double_ml
>>> result = double_ml(data.Y, data.T, data.X)

class dml_ts.data.oj_loader.OJDataLoader[source]#

Bases: object

Loader for Dominick’s Orange Juice dataset.

Downloads and preprocesses the OJ dataset for DML price elasticity estimation. Supports caching to avoid repeated downloads.

Parameters:

  • cache_dir (Union[Path, str, None]) – Directory to cache downloaded data. Defaults to None (no caching).

  • features (Optional[list[str]]) – Confounders to include. Defaults to [“feat”, “INCOME”, “AGE60”].

  • brand (Optional[str]) – Filter to specific brand (“dominicks”, “minute.maid”, “tropicana”). Defaults to None (all brands).

Examples

>>> data = loader.load()
>>> print(data)
OJDataset(n_samples=28947, n_features=3, features=['feat', 'INCOME', 'AGE60'])

>>> # Custom features
>>> loader = OJDataLoader(features=["feat", "INCOME", "AGE60", "EDUC"])
>>> data = loader.load()

FRED Loader (dml_ts.data.fred_loader)#

FRED (Federal Reserve Economic Data) macroeconomic series loader with stationarity transforms and synthetic data generation for offline use.

FRED Data Loader for Macroeconomic Controls in DML.

Provides standardized access to Federal Reserve Economic Data (FRED) for use as control variables in time series DML applications.

Key Features: 1. Automated retrieval of macro indicators 2. Time alignment and frequency conversion 3. Missing data handling with interpolation 4. Caching to avoid repeated API calls

Common Use Cases: - GDP growth as business cycle control - CPI/inflation as price level control - Interest rates for monetary policy studies - Unemployment for labor market analysis

References: - FRED API: https://fred.stlouisfed.org/docs/api/fred/

Usage:
>>> from dml_ts.data import FREDLoader
>>> loader = FREDLoader(api_key="your_api_key")  # or from env
>>> macro_data = loader.get_macro_controls(
...     start_date="2010-01-01",
...     end_date="2020-12-31",
...     frequency="M"
... )

dml_ts.data.fred_loader.create_synthetic_fred_data(start_date='2010-01-01', end_date='2023-12-31', frequency='M', seed=None)[source]#

Create synthetic FRED-like data for testing without API access.

Generates realistic macro data with: - GDP growth: AR(1) with trend - Inflation: AR(1) with mean reversion - Unemployment: Counter-cyclical to GDP - Interest rates: Taylor rule approximation

Parameters:

  • start_date (str) – Start date

  • end_date (str) – End date

  • frequency (Literal['D', 'W', 'M', 'Q', 'A']) – Data frequency

  • seed (Optional[int]) – Random seed for reproducibility

Return type:

MacroControlsResult

Returns:

MacroControlsResult with synthetic macro data

Example

>>> data = create_synthetic_fred_data(seed=42)
>>> X_macro = data.data.values  # Use as controls in DML
