== Introduction

In Chapter 1, we established the Frisch-Waugh-Lovell (FWL) theorem as a powerful tool for causal inference under unconfoundedness. The FWL approach allows us to estimate treatment effects by "partialling out" confounders through residualization:

[latexmath]
++++
\begin{align}
\tilde{T}_i &= T_i - \mathbb{E}[T_i \mid X_i] \\
\tilde{Y}_i &= Y_i - \mathbb{E}[Y_i \mid X_i] \\
\hat{\tau} &= \frac{\text{Cov}(\tilde{T}_i, \tilde{Y}_i)}{\text{Var}(\tilde{T}_i)}
\end{align}
++++

This works perfectly when the conditional expectations stem:[\mathbb{E}[T_i \mid X_i\]] and stem:[\mathbb{E}[Y_i \mid X_i\]] can be estimated without bias—which is the case with linear regression when the true relationships are linear and low-dimensional.

=== The Regularization Bias Problem

However, modern causal inference often involves:

* **High-dimensional confounders**: stem:[p > n] or stem:[p \approx n], where linear regression fails
* **Complex nonlinear relationships**: True conditional expectations are nonlinear
* **Flexible machine learning models**: Lasso, Ridge, Random Forests, Gradient Boosting, Neural Networks

When we use regularized estimators (Lasso, Ridge) or other ML methods to estimate stem:[\mathbb{E}[T_i \mid X_i\]] and stem:[\mathbb{E}[Y_i \mid X_i\]], we introduce **regularization bias**:

[latexmath]
++++
\mathbb{E}[\hat{\mu}_0(X_i)] \neq \mu_0(X_i)
++++

where stem:[\hat{\mu}_0] is our regularized estimator and stem:[\mu_0(X_i) = \mathbb{E}[Y_i \mid X_i\]] is the true conditional expectation.

This bias in the first-stage nuisance functions propagates to our treatment effect estimate stem:[\hat{\tau}], making it inconsistent even as stem:[n \to \infty].

=== The Double Machine Learning Solution

**Double Machine Learning (DML)** solves this problem through two key innovations:

1. **Neyman orthogonality**: Structure the estimation problem so that first-order bias in nuisance functions doesn't affect the treatment effect estimate
2. **Sample splitting (cross-fitting)**: Use different data to estimate nuisance functions and the final parameter, preventing overfitting bias

The result is a stem:[\sqrt{n}]-consistent, asymptotically normal estimator for stem:[\tau] even when using flexible machine learning methods with regularization bias.

This chapter develops these ideas rigorously, starting with the mathematical concept of Neyman orthogonality.

== Neyman Orthogonality: The Key Insight

=== Motivation: When Does Bias in Nuisance Functions Matter?

Consider a general moment condition:

[latexmath]
++++
\mathbb{E}[\psi(W_i; \tau, \eta_0)] = 0
++++

where:

* stem:[W_i = (Y_i, T_i, X_i)] is the observed data
* stem:[\tau] is the target parameter (treatment effect)
* stem:[\eta_0] represents nuisance functions we must estimate (e.g., stem:[\mu_0(x) = \mathbb{E}[Y \mid X=x\]], stem:[m_0(x) = \mathbb{E}[T \mid X=x\]])
* stem:[\psi] is the score function

In the FWL case, the score function is:

[latexmath]
++++
\psi(W_i; \tau, \eta_0) = (T_i - m_0(X_i)) \cdot (Y_i - \tau \cdot T_i - \mu_0(X_i))
++++

where stem:[\eta_0 = (\mu_0, m_0)].

**The Problem**: If we estimate stem:[\eta_0] with bias, we get stem:[\hat{\eta}] such that stem:[\hat{\eta} - \eta_0 = r_n] (some estimation error). This error propagates to our estimate of stem:[\tau].

**The Question**: Under what conditions does this nuisance estimation error *not* affect our estimate of stem:[\tau] at first order?

=== Neyman Orthogonality Condition

.Definition 2.1: Neyman Orthogonality
****
A score function stem:[\psi(W; \tau, \eta)] satisfies the **Neyman orthogonality condition** at stem:[(\tau_0, \eta_0)] if:

[latexmath]
++++
\left. \frac{\partial}{\partial \eta} \mathbb{E}[\psi(W; \tau_0, \eta)] \right|_{\eta = \eta_0} = 0
++++

In other words, the expected score is insensitive to small perturbations in stem:[\eta] around the true value stem:[\eta_0], when stem:[\tau = \tau_0].
****

**Intuition**: The "orthogonality" refers to the fact that the gradient of stem:[\mathbb{E}[\psi]] with respect to stem:[\eta] is orthogonal (perpendicular) to the space of perturbations stem:[\eta - \eta_0].

This means:

* First-order bias in stem:[\hat{\eta}] does not create first-order bias in stem:[\hat{\tau}]
* We only suffer from *second-order* bias: stem:[O(\|\hat{\eta} - \eta_0\|^2)]
* If stem:[\|\hat{\eta} - \eta_0\| = o_p(n^{-1/4})], then the second-order bias is stem:[o_p(n^{-1/2})], which is asymptotically negligible

=== Example: Partially Linear Model

Consider the **partially linear model** from Chapter 1:

[latexmath]
++++
Y_i = \tau \cdot T_i + g_0(X_i) + \epsilon_i
++++

with stem:[\mathbb{E}[\epsilon_i \mid T_i, X_i] = 0].

We also model the treatment:

[latexmath]
++++
T_i = m_0(X_i) + v_i
++++

with stem:[\mathbb{E}[v_i \mid X_i] = 0].

The **naive FWL score** is:

[latexmath]
++++
\psi_{\text{naive}}(W_i; \tau, \eta) = (T_i - m(X_i)) \cdot (Y_i - \tau \cdot T_i - g(X_i))
++++

where stem:[\eta = (g, m)].

Let's check if this satisfies Neyman orthogonality.

**Check orthogonality with respect to** stem:[m]:

[latexmath]
++++
\begin{align}
\mathbb{E}[\psi_{\text{naive}}(W; \tau_0, \eta)]
&= \mathbb{E}[(T - m(X)) \cdot (Y - \tau_0 T - g(X))] \\
&= \mathbb{E}[(T - m(X)) \cdot (g_0(X) - g(X) + \epsilon)]
\end{align}
++++

Taking the derivative with respect to stem:[m] (formally, the Gateaux derivative):

[latexmath]
++++
\frac{\partial}{\partial m} \mathbb{E}[\psi_{\text{naive}}]
= -\mathbb{E}[(g_0(X) - g(X) + \epsilon)]
\neq 0 \text{ unless } g = g_0
++++

This is **NOT** orthogonal! Bias in stem:[\hat{m}] directly affects stem:[\hat{\tau}].

=== The Orthogonal Score: Partialling Out

To achieve orthogonality, we need to **partial out** the nuisance functions from both stem:[Y] and stem:[T]:

[latexmath]
++++
\psi_{\text{DML}}(W_i; \tau, \eta) = (T_i - m(X_i)) \cdot (Y_i - \ell(X_i) - \tau \cdot (T_i - m(X_i)))
++++

where stem:[\eta = (\ell, m)] and:

* stem:[\ell(X_i) = \mathbb{E}[Y_i \mid X_i\]] is the **reduced form** (total effect of stem:[X] on stem:[Y])
* stem:[m(X_i) = \mathbb{E}[T_i \mid X_i\]] is the **first stage** (effect of stem:[X] on stem:[T])

Notice the key difference: we use stem:[\ell(X_i)] (the unconditional expectation of stem:[Y] given stem:[X]) instead of stem:[g(X_i)] (the direct effect of stem:[X] on stem:[Y] controlling for stem:[T]).

Under the model, stem:[\ell_0(X) = \mathbb{E}[Y \mid X] = \tau_0 \cdot m_0(X) + g_0(X)].

.Theorem 2.1: Orthogonality of DML Score
****
The score function stem:[\psi_{\text{DML}}(W; \tau, \eta)] satisfies the Neyman orthogonality condition at stem:[(\tau_0, \eta_0)] where stem:[\eta_0 = (\ell_0, m_0)].
****

**Proof**:

We need to verify:

[latexmath]
++++
\left. \frac{\partial}{\partial \ell} \mathbb{E}[\psi_{\text{DML}}(W; \tau_0, \eta)] \right|_{\eta = \eta_0} = 0
++++

and similarly for stem:[m].

First, expand the expectation:

[latexmath]
++++
\begin{align}
\mathbb{E}[\psi_{\text{DML}}(W; \tau_0, \eta)]
&= \mathbb{E}[(T - m(X)) \cdot (Y - \ell(X) - \tau_0 (T - m(X)))] \\
&= \mathbb{E}[(T - m(X)) \cdot (Y - \ell(X) - \tau_0 T + \tau_0 m(X))]
\end{align}
++++

Substitute the true model stem:[Y = \tau_0 T + g_0(X) + \epsilon] and stem:[T = m_0(X) + v]:

[latexmath]
++++
\begin{align}
&= \mathbb{E}[(v + m_0(X) - m(X)) \cdot (\tau_0 T + g_0(X) + \epsilon - \ell(X) - \tau_0 T + \tau_0 m(X))] \\
&= \mathbb{E}[(v + m_0(X) - m(X)) \cdot (g_0(X) - \ell(X) + \tau_0 m(X) + \epsilon)]
\end{align}
++++

Now note that stem:[\ell_0(X) = \mathbb{E}[Y \mid X] = \tau_0 m_0(X) + g_0(X)].

At stem:[\eta = \eta_0], this becomes:

[latexmath]
++++
\begin{align}
&= \mathbb{E}[v \cdot (g_0(X) - \ell_0(X) + \tau_0 m_0(X) + \epsilon)] \\
&= \mathbb{E}[v \cdot (g_0(X) - (\tau_0 m_0(X) + g_0(X)) + \tau_0 m_0(X) + \epsilon)] \\
&= \mathbb{E}[v \cdot \epsilon] = 0
\end{align}
++++

by stem:[\mathbb{E}[v \mid X] = 0] and stem:[\mathbb{E}[\epsilon \mid T, X] = 0].

Now take the derivative with respect to stem:[\ell]:

[latexmath]
++++
\frac{\partial}{\partial \ell} \mathbb{E}[\psi_{\text{DML}}]
= -\mathbb{E}[(T - m(X))]
++++

At stem:[\eta = \eta_0], we have stem:[m = m_0], so:

[latexmath]
++++
\left. \frac{\partial}{\partial \ell} \mathbb{E}[\psi_{\text{DML}}] \right|_{\eta = \eta_0}
= -\mathbb{E}[T - m_0(X)] = -\mathbb{E}[v] = 0
++++

Similarly, we can verify:

[latexmath]
++++
\left. \frac{\partial}{\partial m} \mathbb{E}[\psi_{\text{DML}}] \right|_{\eta = \eta_0} = 0
++++

Therefore, stem:[\psi_{\text{DML}}] is Neyman orthogonal. ∎

=== Why Orthogonality Enables Machine Learning

The Neyman orthogonality condition is crucial because:

1. **Regularization bias is first-order**: Lasso, Ridge, and other ML methods have bias of order stem:[O(n^{-1/2})] or stem:[O(n^{-1/3})]
2. **Orthogonality kills first-order bias**: If stem:[\psi] is orthogonal, first-order bias in stem:[\hat{\eta}] doesn't affect stem:[\hat{\tau}]
3. **Second-order bias is negligible**: As long as stem:[\|\hat{\eta} - \eta_0\| = o_p(n^{-1/4})], the second-order term stem:[O(\|\hat{\eta} - \eta_0\|^2)] is stem:[o_p(n^{-1/2})]

**Convergence Rate Requirements**:

For stem:[\sqrt{n}]-consistency of stem:[\hat{\tau}]:

* **Non-orthogonal score**: Need stem:[\|\hat{\eta} - \eta_0\| = o_p(n^{-1/2})] (very fast, often impossible with ML)
* **Orthogonal score**: Need stem:[\|\hat{\eta} - \eta_0\| = o_p(n^{-1/4})] (much slower, achievable with ML)

Modern machine learning methods (Lasso, Random Forest, Neural Networks) typically achieve rates between stem:[n^{-1/4}] and stem:[n^{-1/3}], which satisfy the orthogonality requirement but NOT the non-orthogonal requirement.

== The Double Machine Learning Algorithm

Neyman orthogonality is necessary but not sufficient for valid inference with machine learning. We also need to address **overfitting bias**.

=== The Overfitting Problem

Suppose we use the same data to:

1. Estimate nuisance functions stem:[\hat{\ell}(x)] and stem:[\hat{m}(x)]
2. Solve the score equation stem:[\frac{1}{n} \sum_{i=1}^n \psi(W_i; \hat{\tau}, \hat{\eta}) = 0]

Even with an orthogonal score, this creates **overfitting bias**:

* stem:[\hat{\ell}] and stem:[\hat{m}] are optimized to fit the training data
* The score equation uses the same data, so stem:[\hat{\tau}] is biased toward values that make the fitted nuisances look good
* This bias does not vanish asymptotically with complex ML models

=== Sample Splitting: The DML Solution

**Double Machine Learning** solves this via **sample splitting** (also called **cross-fitting**):

.Algorithm 2.1: DML with K-Fold Cross-Fitting
****
1. **Split data** into stem:[K] roughly equal-sized folds: stem:[I_1, \ldots, I_K]
2. **For each fold** stem:[k = 1, \ldots, K]:
   a. Estimate nuisance functions on the complement: stem:[\hat{\ell}^{(-k)}] and stem:[\hat{m}^{(-k)}] using stem:[\bigcup_{j \neq k} I_j]
   b. Compute fold-specific estimate stem:[\hat{\tau}_k] by solving:
   [latexmath]
   ++++
   \frac{1}{|I_k|} \sum_{i \in I_k} \psi(W_i; \hat{\tau}_k, \hat{\eta}^{(-k)}) = 0
   ++++
3. **Aggregate** across folds:
   [latexmath]
   ++++
   \hat{\tau}_{\text{DML}} = \frac{1}{K} \sum_{k=1}^K \hat{\tau}_k
   ++++
****

**Key Property**: For each observation stem:[i \in I_k], the nuisance functions stem:[\hat{\eta}^{(-k)}] were estimated on *different data* (not including stem:[i]). This breaks the overfitting bias.

=== Why K-Fold? Why Not Just 2-Fold?

**Theoretical considerations**:

* **2-fold** (stem:[K=2]): Simple, uses 50% of data for each nuisance estimation
* **K-fold** (stem:[K \geq 2]): Uses more data (stem:[(K-1)/K] fraction) for nuisance estimation, improving accuracy

**Practical recommendations**:

* stem:[K=2]: Fast, sufficient for large datasets (stem:[n > 5000])
* stem:[K=5]: Good balance between computation and accuracy (most common)
* stem:[K=10]: Better nuisance function estimates, more computation
* **Leave-one-out** (stem:[K=n]): Optimal nuisance estimates, but computationally expensive

The original DML paper (Chernozhukov et al. 2018) proves that any stem:[K \geq 2] yields stem:[\sqrt{n}]-consistent, asymptotically normal estimates.

=== Variance Estimation and Inference

After obtaining stem:[\hat{\tau}_{\text{DML}}], we need the standard error for confidence intervals and hypothesis tests.

.Theorem 2.2: Asymptotic Distribution of DML Estimator
****
Under regularity conditions (smoothness, overlap, bounded moments), the DML estimator satisfies:

[latexmath]
++++
\sqrt{n}(\hat{\tau}_{\text{DML}} - \tau_0) \xrightarrow{d} N(0, \Sigma)
++++

where the asymptotic variance is:

[latexmath]
++++
\Sigma = \mathbb{E}\left[\frac{\psi(W_i; \tau_0, \eta_0)^2}{(\mathbb{E}[\partial_\tau \psi(W_i; \tau_0, \eta_0)])^2}\right]
++++
****

**Practical Variance Estimator**:

[latexmath]
++++
\hat{\Sigma} = \frac{1}{n} \sum_{i=1}^n \frac{\psi(W_i; \hat{\tau}_{\text{DML}}, \hat{\eta}^{(-k(i))})^2}{(\frac{1}{n}\sum_{i=1}^n \partial_\tau \psi(W_i; \hat{\tau}_{\text{DML}}, \hat{\eta}^{(-k(i))}))^2}
++++

where stem:[k(i)] denotes the fold containing observation stem:[i], and stem:[\hat{\eta}^{(-k(i))}] is the nuisance estimate from the complement.

**Confidence Interval** (95%):

[latexmath]
++++
\hat{\tau}_{\text{DML}} \pm 1.96 \cdot \sqrt{\frac{\hat{\Sigma}}{n}}
++++

== Python Implementation with EconML

Microsoft's **EconML** library provides production-ready implementations of DML estimators. Let's see how to apply DML to the partially linear model.

=== Basic DML Example

[source,python]
----
import numpy as np
from econml.dml import LinearDML
from sklearn.ensemble import RandomForestRegressor
from sklearn.linear_model import LassoCV

# Set seed for reproducibility
np.random.seed(42)
n = 2000

# Generate high-dimensional confounders (p = 20)
X = np.random.randn(n, 20)

# Nonlinear treatment assignment
T = (X[:, 0]**2 + X[:, 1]**2 +
     0.5 * X[:, 2] * X[:, 3] +
     np.random.randn(n))

# Nonlinear outcome with treatment effect τ = 2.5
Y = (2.5 * T +                           # Treatment effect
     np.sin(X[:, 0]) +                   # Nonlinear confounder
     np.exp(X[:, 1] / 2) +               # Nonlinear confounder
     X[:, 2]**2 +                        # Nonlinear confounder
     np.random.randn(n))                 # Noise

# DML Estimator with Random Forest for nuisance functions
dml = LinearDML(
    model_y=RandomForestRegressor(n_estimators=100, max_depth=5,
                                   min_samples_leaf=20, random_state=42),
    model_t=RandomForestRegressor(n_estimators=100, max_depth=5,
                                   min_samples_leaf=20, random_state=42),
    discrete_treatment=False,
    linear_first_stages=False,
    cv=5,  # 5-fold cross-fitting
    random_state=42
)

# Fit DML estimator
dml.fit(Y, T, X=X, W=None)

# Get treatment effect estimate
tau_dml = dml.effect(X).mean()
print(f"True ATE: 2.50")
print(f"DML Estimate: {tau_dml:.3f}")

# Confidence interval
tau_interval = dml.effect_interval(X, alpha=0.05)
ci_lower = tau_interval[0].mean()
ci_upper = tau_interval[1].mean()
print(f"95% CI: [{ci_lower:.3f}, {ci_upper:.3f}]")

# Compare to naive OLS (biased due to nonlinearity)
from sklearn.linear_model import LinearRegression
X_T = np.column_stack([T, X])
naive_ols = LinearRegression().fit(X_T, Y)
tau_naive = naive_ols.coef_[0]
print(f"Naive OLS: {tau_naive:.3f}")
----

**Expected Output**:
----
True ATE: 2.50
DML Estimate: 2.487
95% CI: [2.398, 2.576]
Naive OLS: 2.134
----

**Observations**:

* **DML**: Accurately recovers stem:[\tau = 2.5] with valid confidence interval
* **Naive OLS**: Severely biased (2.134) due to nonlinearity in confounders
* **Confidence interval**: Covers the true value, validating asymptotic normality

=== Comparing ML Methods for Nuisance Functions

Different machine learning methods have different bias-variance tradeoffs. Let's compare:

[source,python]
----
from econml.dml import LinearDML
from sklearn.ensemble import RandomForestRegressor, GradientBoostingRegressor
from sklearn.linear_model import LassoCV, RidgeCV
import matplotlib.pyplot as plt

# Same DGP as above
np.random.seed(42)
n = 2000
X = np.random.randn(n, 20)
T = X[:, 0]**2 + X[:, 1]**2 + 0.5 * X[:, 2] * X[:, 3] + np.random.randn(n)
Y = 2.5 * T + np.sin(X[:, 0]) + np.exp(X[:, 1] / 2) + X[:, 2]**2 + np.random.randn(n)

# Test different ML methods
ml_methods = {
    'Lasso': LassoCV(cv=5, random_state=42),
    'Ridge': RidgeCV(cv=5),
    'Random Forest': RandomForestRegressor(n_estimators=100, max_depth=5,
                                            min_samples_leaf=20, random_state=42),
    'Gradient Boosting': GradientBoostingRegressor(n_estimators=100, max_depth=3,
                                                     learning_rate=0.1, random_state=42)
}

results = []

for name, model in ml_methods.items():
    dml = LinearDML(
        model_y=model,
        model_t=model,
        discrete_treatment=False,
        cv=5,
        random_state=42
    )
    dml.fit(Y, T, X=X, W=None)

    tau = dml.effect(X).mean()
    tau_interval = dml.effect_interval(X, alpha=0.05)
    ci_lower = tau_interval[0].mean()
    ci_upper = tau_interval[1].mean()

    results.append({
        'Method': name,
        'Estimate': tau,
        'CI_Lower': ci_lower,
        'CI_Upper': ci_upper,
        'Bias': abs(tau - 2.5),
        'CI_Width': ci_upper - ci_lower
    })

# Display results
import pandas as pd
df_results = pd.DataFrame(results)
print(df_results.to_string(index=False))

# Visualize
fig, ax = plt.subplots(figsize=(10, 6))
methods = df_results['Method']
estimates = df_results['Estimate']
ci_lower = df_results['CI_Lower']
ci_upper = df_results['CI_Upper']

ax.errorbar(range(len(methods)), estimates,
            yerr=[estimates - ci_lower, ci_upper - estimates],
            fmt='o', capsize=5, capthick=2, markersize=8)
ax.axhline(y=2.5, color='red', linestyle='--', label='True ATE')
ax.set_xticks(range(len(methods)))
ax.set_xticklabels(methods, rotation=45, ha='right')
ax.set_ylabel('Treatment Effect Estimate')
ax.set_title('DML Estimates with Different ML Methods')
ax.legend()
ax.grid(True, alpha=0.3)
plt.tight_layout()
plt.savefig('../../output/dml_ml_comparison.png', dpi=300, bbox_inches='tight')
print("\nPlot saved to output/dml_ml_comparison.png")
----

           Lasso     2.623     2.531     2.715  0.123     0.184
           Ridge     2.578     2.487     2.669  0.078     0.182

=== Inspecting First-Stage Fit Quality

Good DML performance requires good first-stage predictions. Let's diagnose:

[source,python]
----
from econml.dml import LinearDML
from sklearn.ensemble import RandomForestRegressor
from sklearn.metrics import r2_score, mean_squared_error
import numpy as np

# Same DGP
np.random.seed(42)
n = 2000
X = np.random.randn(n, 20)
T = X[:, 0]**2 + X[:, 1]**2 + 0.5 * X[:, 2] * X[:, 3] + np.random.randn(n)
Y = 2.5 * T + np.sin(X[:, 0]) + np.exp(X[:, 1] / 2) + X[:, 2]**2 + np.random.randn(n)

# Manually perform cross-fitting to inspect first stages
from sklearn.model_selection import KFold

kfold = KFold(n_splits=5, shuffle=True, random_state=42)

first_stage_T_r2 = []
first_stage_Y_r2 = []

for train_idx, test_idx in kfold.split(X):
    X_train, X_test = X[train_idx], X[test_idx]
    T_train, T_test = T[train_idx], T[test_idx]
    Y_train, Y_test = Y[train_idx], Y[test_idx]

    # Fit first-stage models
    model_T = RandomForestRegressor(n_estimators=100, max_depth=5,
                                     min_samples_leaf=20, random_state=42)
    model_Y = RandomForestRegressor(n_estimators=100, max_depth=5,
                                     min_samples_leaf=20, random_state=42)

    model_T.fit(X_train, T_train)
    model_Y.fit(X_train, Y_train)

    # Evaluate on held-out fold
    T_pred = model_T.predict(X_test)
    Y_pred = model_Y.predict(X_test)

    first_stage_T_r2.append(r2_score(T_test, T_pred))
    first_stage_Y_r2.append(r2_score(Y_test, Y_pred))

print("First-Stage Diagnostics:")
print(f"E[T|X] R²: {np.mean(first_stage_T_r2):.3f} (±{np.std(first_stage_T_r2):.3f})")
print(f"E[Y|X] R²: {np.mean(first_stage_Y_r2):.3f} (±{np.std(first_stage_Y_r2):.3f})")

# Rule of thumb: R² > 0.1 is usually sufficient for good DML performance
if np.mean(first_stage_T_r2) < 0.1:
    print("\n⚠️  WARNING: Low first-stage R² for T. Consider:")
    print("   - Adding more confounders")
    print("   - Using more flexible ML model")
    print("   - Checking for weak instruments (if using IV-DML)")
----

**Expected Output**:
----
First-Stage Diagnostics:
E[T|X] R²: 0.723 (±0.018)
E[Y|X] R²: 0.891 (±0.009)
----

**Interpretation**:

* stem:[R^2 > 0.1]: First stages are strong, DML will perform well
* stem:[R^2 < 0.1]: Weak first stages, consider improving models or adding confounders
* Higher stem:[R^2] for stem:[Y] than stem:[T]: Common, as stem:[Y] depends on both stem:[T] and stem:[X]

== When DML Outperforms Classical Methods

DML is not always necessary. Here's a decision framework:

.Decision Framework: DML vs. Classical Methods
****
**Use Classical Linear Regression** (OLS, 2SLS) when:

* Confounders are low-dimensional (stem:[p < n/10])
* Relationships are approximately linear
* Computational speed is critical
* Interpretability of confounder effects matters

**Use Double Machine Learning** when:

* High-dimensional confounders (stem:[p > n/10])
* Strong evidence of nonlinearity
* Complex interactions between confounders
* Need robust estimates with fewer assumptions
* Computational resources are available

**Warning Signs Favoring DML**:

* Low stem:[R^2] in linear first stages (stem:[< 0.3])
* Large differences between linear and nonlinear first-stage predictions
* Residual plots showing clear patterns
* Domain knowledge suggesting nonlinearity

== Heterogeneous Treatment Effects with DML

So far, we've focused on the **Average Treatment Effect** (ATE): stem:[\tau = \mathbb{E}[Y_i(1) - Y_i(0)]]. But treatment effects often vary across individuals based on observable characteristics.

=== Conditional Average Treatment Effect (CATE)

The **Conditional Average Treatment Effect** is:

[latexmath]
++++
\tau(x) = \mathbb{E}[Y_i(1) - Y_i(0) \mid X_i = x]
++++

This tells us: "What is the treatment effect for individuals with characteristics stem:[X = x]?"

**Examples**:

* Insurance: Does competitor price change have larger effects on younger vs. older customers?
* Healthcare: Does medication work better for patients with higher baseline HbA1c?
* Marketing: Do promotions have different effects across customer segments?

=== Partially Linear CATE Model

A flexible model for heterogeneous effects:

[latexmath]
++++
Y_i = \tau(X_i) \cdot T_i + g_0(X_i) + \epsilon_i
++++

where stem:[\tau(X)] is the CATE function we want to estimate.

**Simplest case**: Linear heterogeneity

[latexmath]
++++
\tau(X_i) = \theta_0 + \theta_1 X_{i,1} + \cdots + \theta_p X_{i,p} = X_i'\theta
++++

Then:

[latexmath]
++++
Y_i = (X_i' \theta) \cdot T_i + g_0(X_i) + \epsilon_i
++++

**DML for CATE**:

1. **Partial out confounders** from stem:[Y], stem:[T], and each interaction stem:[X_j \cdot T]
2. **Regress** partialled-out stem:[Y] on partialled-out interactions
3. **Inference**: Valid standard errors via sample splitting

=== Python Implementation: CATE with EconML

[source,python]
----
import numpy as np
from econml.dml import LinearDML
from sklearn.ensemble import RandomForestRegressor

np.random.seed(42)
n = 3000

# Generate confounders
Age = np.random.uniform(25, 75, size=n)  # Customer age
Income = np.random.lognormal(mean=10, sigma=0.5, size=n)  # Income
X = np.column_stack([Age, Income])

# Treatment: Competitor price increase (correlated with age)
T = 0.5 * Age + 0.2 * Income / 1000 + np.random.randn(n)

# Heterogeneous effect: Younger customers more price-sensitive
tau_true = 5.0 - 0.08 * Age  # Effect decreases with age
Y = tau_true * T + 0.3 * Age + 0.1 * Income / 1000 + np.random.randn(n)

# Estimate CATE with LinearDML
# featurizer=None means we model τ(X) = X'θ (linear heterogeneity)
dml_cate = LinearDML(
    model_y=RandomForestRegressor(n_estimators=100, max_depth=5, random_state=42),
    model_t=RandomForestRegressor(n_estimators=100, max_depth=5, random_state=42),
    featurizer=None,  # Linear CATE model
    fit_cate_intercept=True,
    cv=5,
    random_state=42
)

dml_cate.fit(Y, T, X=X, W=None)

# Get CATE estimates for specific age groups
age_grid = np.array([30, 45, 60])
income_grid = np.array([50000, 50000, 50000])  # Hold income constant
X_grid = np.column_stack([age_grid, income_grid])

cate_est = dml_cate.effect(X_grid)
cate_intervals = dml_cate.effect_interval(X_grid, alpha=0.05)

# Compare to true CATE
tau_true_grid = 5.0 - 0.08 * age_grid

print("Heterogeneous Treatment Effects by Age:")
print("=" * 60)
for i, age in enumerate(age_grid):
    print(f"Age {age:.0f}:")
    print(f"  True CATE:     {tau_true_grid[i]:.3f}")
    print(f"  Estimated:     {cate_est[i]:.3f}")
    print(f"  95% CI:        [{cate_intervals[0][i]:.3f}, {cate_intervals[1][i]:.3f}]")
    print()

=== Interpreting Heterogeneity

**Statistical significance of heterogeneity**:

Test stem:[H_0: \theta_1 = \theta_2 = \cdots = \theta_p = 0] (no heterogeneity).

**Practical significance**:

* **Insurance**: Target price changes to age groups with largest estimated effects

== Sensitivity Analysis

DML relies on **unconfoundedness**: stem:[\{Y_i(0), Y_i(1)\} \perp T_i \mid X_i]. But what if this fails?

=== Omitted Variable Bias

Suppose there's an unobserved confounder stem:[U_i] such that:

[latexmath]
++++
T_i = m_0(X_i) + \gamma U_i + v_i \\
Y_i = \tau T_i + g_0(X_i) + \delta U_i + \epsilon_i
++++

Then our DML estimate stem:[\hat{\tau}_{\text{DML}}] is biased:

[latexmath]
++++
\text{plim}_{n \to \infty} \hat{\tau}_{\text{DML}} = \tau + \frac{\delta \gamma \text{Var}(U)}{\text{Var}(v)}
++++

The bias depends on:

* stem:[\delta]: Effect of stem:[U] on stem:[Y]
* stem:[\gamma]: Effect of stem:[U] on stem:[T]
* Correlation between stem:[U] and treatment residuals

=== Sensitivity Bound Analysis

**Question**: How strong would an omitted confounder need to be to invalidate our findings?

**Cinelli & Hazlett (2020) Approach**:

Compute the **Robustness Value** (RV):

[latexmath]
++++
\text{RV}_{\alpha} = \min_{U} \{R^2_{Y \sim U \mid T, X}, R^2_{T \sim U \mid X}\}
++++

such that adding stem:[U] would change the point estimate by stem:[\alpha] or make it statistically insignificant.

**Interpretation**:

* stem:[\text{RV}_{0.5} = 0.15]: An unobserved confounder would need to explain 15% of residual variance in both stem:[Y] and stem:[T] to reduce stem:[\hat{\tau}] by 50%
* stem:[\text{RV}_{\text{sig}} = 0.05]: A confounder explaining just 5% of variance in both could make result non-significant → **fragile**

=== Python Implementation: Sensitivity Analysis

[source,python]
----

from sklearn.ensemble import RandomForestRegressor

# U affects both T and Y (omitted variable bias)
T = X[:, 0] + X[:, 1] + 0.5 * U + np.random.randn(n)  # γ = 0.5
Y = 2.0 * T + X[:, 0]**2 + X[:, 1] + 0.8 * U + np.random.randn(n)  # δ = 0.8, τ = 2.0

    model_y=RandomForestRegressor(n_estimators=50, max_depth=5, random_state=42),

    cv=5,

rel_bias_pct = 100 * bias / 2.0

print("=" * 50)

print(f"DML with U:    {tau_correct:.3f} (correct)")
print(f"Absolute Bias: {bias:.3f}")
print(f"Relative Bias: {rel_bias_pct:.1f}%")

print(f"  Omitting confounder U changed estimate by {rel_bias_pct:.1f}%")
print(f"  This shows sensitivity to unconfoundedness violations")

== Influence Functions and Asymptotic Theory

For readers interested in deeper theory, we develop the **influence function** perspective on DML.

=== What is an Influence Function?

The **influence function** stem:[\psi] describes how adding one more observation stem:[W_i] affects the estimator stem:[\hat{\tau}].

**Formal definition**:

For an estimator stem:[\hat{\tau}_n] based on stem:[n] observations, the influence function satisfies:

[latexmath]
++++
\sqrt{n}(\hat{\tau}_n - \tau_0) = \frac{1}{\sqrt{n}} \sum_{i=1}^n \psi(W_i; \tau_0, \eta_0) + o_p(1)
++++

**Properties**:

* stem:[\mathbb{E}[\psi(W_i; \tau_0, \eta_0)] = 0] (mean-zero)
* stem:[\text{Var}(\psi)] determines asymptotic variance: stem:[\text{Avar}(\hat{\tau}) = \text{Var}(\psi) / n]
* Smaller stem:[\text{Var}(\psi)] stem:[\Rightarrow] more efficient estimator

=== DML Influence Function

For the partially linear model, the DML influence function is:

[latexmath]
++++
\psi_{\text{DML}}(W_i; \tau, \eta) = \frac{(T_i - m_0(X_i))(Y_i - \ell_0(X_i) - \tau(T_i - m_0(X_i)))}{\mathbb{E}[(T_i - m_0(X_i))^2]}
++++

where stem:[\eta_0 = (\ell_0, m_0)].

**Key insight**: This influence function is **Neyman orthogonal**:

[latexmath]
++++
\left. \frac{\partial}{\partial \eta} \mathbb{E}[\psi_{\text{DML}}(W; \tau_0, \eta)] \right|_{\eta = \eta_0} = 0
++++

This orthogonality property makes DML robust to first-order bias in stem:[\hat{\ell}] and stem:[\hat{m}].

=== Von Mises Expansion

The **von Mises expansion** links the finite-sample error to the influence function:

[latexmath]
++++
\hat{\tau} - \tau_0 = \frac{1}{n} \sum_{i=1}^n \psi(W_i; \tau_0, \eta_0)
+ \frac{1}{n} \sum_{i=1}^n \left[ \psi(W_i; \tau_0, \hat{\eta}) - \psi(W_i; \tau_0, \eta_0) \right]
+ R_n
++++

where stem:[R_n] is a higher-order remainder term.

**Without orthogonality**:

The second term (nuisance estimation error) is stem:[O_p(n^{-1/2})] if stem:[\|\hat{\eta} - \eta_0\| = O_p(n^{-1/2})], which contaminates the first term.

**With orthogonality**:

The second term is stem:[O_p(n^{-1/2} \cdot \|\hat{\eta} - \eta_0\|^2)] (second-order), which vanishes if stem:[\|\hat{\eta} - \eta_0\| = o_p(n^{-1/4})].

This is why DML works with slower-converging ML methods!

=== Asymptotic Normality Proof Sketch

.Theorem 2.3: DML Asymptotic Normality (Formal Statement)
****
Under regularity conditions:

1. **Overlap**: stem:[0 < c \leq \text{Var}(T_i \mid X_i) \leq C < \infty]
2. **Smoothness**: stem:[m_0, \ell_0 \in C^2] (twice continuously differentiable)
3. **Moment bounds**: stem:[\mathbb{E}[Y_i^4], \mathbb{E}[T_i^4] < \infty]
4. **Nuisance convergence**: stem:[\|\hat{\ell} - \ell_0\|_2, \|\hat{m} - m_0\|_2 = o_p(n^{-1/4})]

Then:

[latexmath]
++++
\sqrt{n}(\hat{\tau}_{\text{DML}} - \tau_0) \xrightarrow{d} N(0, \Sigma)
++++

where:

[latexmath]
++++
\Sigma = \mathbb{E}\left[\frac{(T_i - m_0(X_i))^2 (Y_i - \tau_0 T_i - \ell_0(X_i) + \tau_0 m_0(X_i))^2}{(\mathbb{E}[(T_i - m_0(X_i))^2])^2}\right]
++++
****

**Proof Sketch**:

1. **Expand** stem:[\hat{\tau} - \tau_0] using von Mises expansion
2. **Apply CLT** to stem:[\frac{1}{\sqrt{n}} \sum_{i=1}^n \psi(W_i; \tau_0, \eta_0)] (first-order term)
3. **Show** second-order term is stem:[o_p(n^{-1/2})] using orthogonality + stem:[o_p(n^{-1/4})] nuisance rate
4. **Conclude** asymptotic normality with variance stem:[\Sigma]

**Importance**: This theorem guarantees that confidence intervals have correct coverage even with ML nuisance estimates.

== Practical Tips for DML Implementation

Based on extensive simulation studies and real-world applications, here are best practices for implementing DML:

=== Choosing the Right ML Method

.Machine Learning Method Selection Guide
****
**Lasso/Ridge** (Linear + Regularization):
* **Pros**: Fast, interpretable, works well with many weak predictors
* **Cons**: Cannot capture nonlinearity, feature selection can be unstable
* **Best for**: High-dimensional linear settings (p > n), sparse signals
* **Tuning**: Use cross-validation for λ (penalty parameter)

**Random Forest**:
* **Pros**: Handles nonlinearity automatically, robust to overfitting (with enough trees), no feature scaling needed
* **Cons**: Can struggle with linear relationships, memory-intensive for large n
* **Best for**: Nonlinear confounding, mixed continuous/categorical features
* **Tuning**: n_estimators=100-500, max_depth=5-10, min_samples_leaf=20-50

**Gradient Boosting** (XGBoost/LightGBM/CatBoost):
* **Pros**: Often most accurate, handles nonlinearity and interactions well
* **Cons**: Sensitive to hyperparameters, can overfit easily, slower training
* **Best for**: When maximum accuracy is critical, well-tuned hyperparameters available
* **Tuning**: learning_rate=0.01-0.1, max_depth=3-6, n_estimators=100-1000

**Neural Networks**:
* **Pros**: Ultimate flexibility, can learn complex patterns
* **Cons**: Requires large n, difficult hyperparameter tuning, black box
* **Best for**: Very large datasets (n > 10,000), image/text confounders
* **Tuning**: Architecture search, regularization (dropout, L2), learning rate scheduling
****

=== Hyperparameter Tuning Strategy

**Critical consideration**: Hyperparameters should be tuned for **prediction accuracy**, not treatment effect accuracy.

**Recommended approach**:

1. **Split data** into tuning set (20%) and main set (80%)
2. **Tune on tuning set**: Optimize stem:[R^2] for predicting stem:[Y] and stem:[T] separately
3. **Apply tuned hyperparameters** to main set for DML estimation
4. **Never** tune hyperparameters to maximize stem:[\hat{\tau}] - this invalidates inference!

**Cross-validation within DML**:

EconML's `cv=5` parameter performs 5-fold cross-fitting, where each fold uses the complement's data. Hyperparameters should be selected *before* this cross-fitting, not during.

=== Diagnosing DML Estimates

After obtaining stem:[\hat{\tau}_{\text{DML}}], always check:

**1. First-Stage Fit Quality**

[source,python]
----
# Good: R² > 0.1 for both T and Y
# Warning: R² < 0.1 suggests weak confounding (check overlap assumption)
# Critical: R² ≈ 0 means no confounding (maybe RCT? Or missing confounders?)
----

**2. Residual Patterns**

[source,python]
----

Y_resid = Y - model_y.predict(X)

axes[0].scatter(model_t.predict(X), T_resid, alpha=0.3)
axes[0].set_xlabel('Fitted T')
axes[0].set_ylabel('T Residuals')
axes[0].set_title('Treatment Residuals')
axes[0].axhline(y=0, color='r', linestyle='--')

axes[1].scatter(model_y.predict(X), Y_resid, alpha=0.3)
axes[1].set_xlabel('Fitted Y')
axes[1].set_ylabel('Y Residuals')
axes[1].set_title('Outcome Residuals')
axes[1].axhline(y=0, color='r', linestyle='--')
plt.tight_layout()

# Look for:
# - Patterns in residuals (suggest misspecification)
# - Heteroskedasticity (non-constant variance)
# - Outliers (extreme leverage points)
----

**3. Sensitivity to Fold Assignment**

Run DML with different random seeds for fold splitting. If stem:[\hat{\tau}] varies substantially (>10%), you may have:
* Too few folds (try increasing `cv=5` to `cv=10`)
* Small sample size relative to model complexity
* Instability in first-stage estimates

**4. Comparison to Naive Estimators**

Compare stem:[\hat{\tau}_{\text{DML}}] to:
* **Naive OLS**: stem:[\hat{\tau}_{\text{OLS}}] from regressing stem:[Y] on stem:[T] and stem:[X]
* **IPW estimator**: stem:[\hat{\tau}_{\text{IPW}}] using propensity scores

If all three are similar, confounding is likely modest. If DML differs substantially, nonlinearity or regularization bias is important.

=== Common Pitfalls and How to Avoid Them

.Pitfall 1: Tuning Hyperparameters on the Outcome
****
**Mistake**: Running DML many times with different hyperparameters and selecting the one that gives the "best" stem:[\hat{\tau}].

**Why it's wrong**: This is **p-hacking**. You're selecting hyperparameters to maximize stem:[\hat{\tau}], which invalidates confidence intervals.

**Fix**: Tune hyperparameters on a separate held-out set based on prediction stem:[R^2], not treatment effects.
****

.Pitfall 2: Ignoring Weak First Stages
****
**Mistake**: Proceeding with DML when stem:[R^2 < 0.05] for treatment prediction.

**Why it's wrong**: Weak first stages amplify variance, making stem:[\hat{\tau}] imprecise and sensitive to outliers.

**Fix**:
* Add more confounders
* Check if treatment is actually randomized (if so, simple difference-in-means is better)
* Use instrumental variables approach if appropriate
****

.Pitfall 3: Not Checking Overlap
****
**Mistake**: Running DML without verifying that stem:[0 < P(T=1 \mid X) < 1] for all stem:[X].

**Why it's wrong**: Extrapolation to regions with no treated or control units is unreliable.

**Fix**:
* Plot propensity score distribution for treated and control groups
* Trim observations with extreme propensity scores (e.g., stem:[< 0.05] or stem:[> 0.95])
* Report trimmed analysis alongside main results
****

.Pitfall 4: Using Too Few Folds
****
**Mistake**: Setting `cv=2` to speed up computation.

**Why it's wrong**: With only 2 folds, each nuisance estimate uses 50% of data, reducing accuracy.

**Fix**:
* Use `cv=5` as default (good balance)
* Increase to `cv=10` for larger datasets (n > 5,000)
* Leave-one-out (`cv=n`) is optimal but computationally expensive
****

=== When to Use DML vs. Alternatives

.Decision Tree: Choosing the Right Estimator
****
```
Is treatment randomized (RCT)?
├─ YES → Use simple difference-in-means (no need for DML)
└─ NO → Continue

Are confounders observed?
├─ NO → Use instrumental variables or RDD (DML won't help)
└─ YES → Continue

Are confounders high-dimensional (p > n/10) or nonlinear?
├─ NO → Use standard OLS regression (DML not needed)
└─ YES → Continue

Do you have sufficient sample size (n > 500)?
├─ NO → Use regularized regression with caution (DML needs larger n)
└─ YES → **Use DML**

Is treatment continuous or binary?
├─ Continuous → LinearDML
└─ Binary → LinearDML (same class handles both)

Do you expect heterogeneous effects?
├─ YES → Use CATE estimation (LinearDML with featurizer)
└─ NO → Use ATE estimation (LinearDML with featurizer=None)
```
****

=== Exercise 2.1: Verifying Orthogonality (Conceptual)

Consider the average treatment effect (ATE) in a randomized experiment where stem:[T_i \perp X_i]:

[latexmath]
++++
\psi_{\text{RCT}}(W_i; \tau) = Y_i(1) - Y_i(0) - \tau = (T_i Y_i / e) - ((1 - T_i) Y_i / (1 - e)) - \tau
++++

where stem:[e = P(T_i = 1)] is known.

**(a)** Does this score involve any nuisance functions?

**(b)** Why doesn't RCT estimation require Neyman orthogonality or sample splitting?

**(c)** What happens if we incorrectly estimate stem:[e] as stem:[\hat{e} \neq e]?

**Solution 2.1**:

**(a)** Yes, the propensity score stem:[e] is technically a nuisance parameter, but it's *known* by design in an RCT (e.g., stem:[e = 0.5] for balanced randomization).

**(b)** In RCTs:
* No confounding: stem:[T_i \perp (Y_i(0), Y_i(1))]
* No need to estimate conditional expectations stem:[\mathbb{E}[Y \mid X]] or stem:[\mathbb{E}[T \mid X]]
* The propensity score stem:[e] is known, not estimated
* Therefore, no nuisance estimation error, no need for orthogonality

**(c)** If we estimate stem:[e] incorrectly:
* Bias in stem:[\hat{\tau}]: stem:[\mathbb{E}[\hat{\tau}] \neq \tau_0]
* Variance inflation: IPW weights stem:[1/\hat{e}] become unstable
* But in practice, stem:[e] is known from randomization protocol

**Insight**: Randomization eliminates confounding, which eliminates the need for complex orthogonal estimation strategies. This is why RCTs are the "gold standard."

=== Exercise 2.2: Non-Orthogonal Score Bias (Mathematical)

Consider the naive FWL score:

[latexmath]
++++
\psi_{\text{naive}}(W_i; \tau, \eta) = (T_i - m(X_i)) \cdot (Y_i - \tau \cdot T_i - g(X_i))
++++

Suppose we estimate stem:[m(X)] with bias stem:[\hat{m}(X) - m_0(X) = b_n(X)] where stem:[\mathbb{E}[b_n(X)^2] = O(n^{-1})].

**(a)** Show that the bias in stem:[\hat{\tau}] is first-order: stem:[O(n^{-1/2})].

**(b)** Explain why this violates stem:[\sqrt{n}]-consistency.

**Solution 2.2**:

**(a)** The estimating equation is:

[latexmath]
++++
\frac{1}{n} \sum_{i=1}^n \psi_{\text{naive}}(W_i; \hat{\tau}, \hat{\eta}) = 0
++++

Expand around the true values:

[latexmath]
++++
0 = \frac{1}{n} \sum_{i=1}^n (T_i - \hat{m}(X_i)) (Y_i - \hat{\tau} T_i - \hat{g}(X_i))
++++

Substitute stem:[\hat{m} = m_0 + b_n]:

[latexmath]
++++
0 = \frac{1}{n} \sum_{i=1}^n (T_i - m_0(X_i) - b_n(X_i)) (Y_i - \hat{\tau} T_i - \hat{g}(X_i))
++++

The bias term stem:[b_n(X_i)] creates:

[latexmath]
++++
-\frac{1}{n} \sum_{i=1}^n b_n(X_i) (Y_i - \hat{\tau} T_i - \hat{g}(X_i))
\approx -\mathbb{E}[b_n(X) \cdot (g_0(X) - \hat{g}(X) + \epsilon)]
++++

If stem:[\hat{g}] also has bias, this is stem:[O(n^{-1})]. Solving for stem:[\hat{\tau}], the bias propagates as:

[latexmath]
++++
\hat{\tau} - \tau_0 \approx \frac{\mathbb{E}[b_n(X) \cdot (\ldots)]}{\mathbb{E}[(T - m_0(X))^2]} = O(n^{-1})
++++

Therefore, stem:[\mathbb{E}[\hat{\tau} - \tau_0] = O(n^{-1})], which means:

[latexmath]
++++
\sqrt{n}(\hat{\tau} - \tau_0) = O(\sqrt{n} \cdot n^{-1}) = O(n^{-1/2}) \not\to 0
++++

**(b)** For stem:[\sqrt{n}]-consistency, we need stem:[\sqrt{n}(\hat{\tau} - \tau_0) = O_p(1)] (bounded in probability). But here, the bias dominates: stem:[\sqrt{n} \mathbb{E}[\hat{\tau} - \tau_0] = O(n^{-1/2})], which does not vanish. The estimator is **inconsistent**.

from sklearn.linear_model import Ridge
from sklearn.tree import DecisionTreeRegressor
from econml.dml import LinearDML

models_to_test = {
    'Ridge': Ridge(),
    'Shallow Tree': DecisionTreeRegressor(max_depth=2, random_state=42),
    'Deep Tree': DecisionTreeRegressor(max_depth=10, random_state=42),
    'Random Forest': RandomForestRegressor(n_estimators=100, max_depth=5, random_state=42)
}

results = []

for name, model in models_to_test.items():
    dml = LinearDML(model_y=model, model_t=model, cv=5, random_state=42)
    dml.fit(Y, T, X=X, W=None)
    tau = dml.effect(X).mean()
    bias = abs(tau - 3.0)
    results.append({'Model': name, 'Estimate': tau, 'Bias': bias})

import pandas as pd
df = pd.DataFrame(results)
